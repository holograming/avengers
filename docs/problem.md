3. 유연한 워크 플로우에 대한 강화
 "캡틴의 요구사항 분석 -> 실행 -> 완료"
 완료라는 의미를 잘 생각할것.
  -> 사용자 요구사항의 완성 기준이 무엇인지를 계획단계에서 확인할것.
  예를 들어 c++ 개발시에는 동작 확인을 하지 않고, 코드만 작성 후 유저한테 확인을 요청하였음.
  실제 실행까지 완료하길 원하는건지. 코드만 작성되길 원하는건지, 문서만 필요한건지.
  완료라는 표현에 많은 가능성이 있을 수 있음. 그러므로 확인 받을것.
