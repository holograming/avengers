# Captain 에이전트 성공 메트릭

> 오케스트레이터(Orchestrator) 역할의 핵심 성과 지표

---

## 1. 작업 분배 정확성 (Task Distribution Accuracy)

### 1.1 역할 매칭 정확도 (Role Matching Accuracy)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 에이전트-작업 적합도 | 작업이 적절한 에이전트에게 할당된 비율 | >= 95% | `(적절한 할당 수 / 전체 할당 수) * 100` |
| 재할당률 (Reassignment Rate) | 할당 후 다른 에이전트로 재할당된 비율 | <= 5% | `(재할당 수 / 전체 할당 수) * 100` |
| 역량 활용도 (Capability Utilization) | 에이전트 전문 역량이 활용된 비율 | >= 90% | 에이전트별 전문 작업 비율 |

### 1.2 작업 분해 품질 (Task Decomposition Quality)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 작업 세분화 적정성 | 너무 크거나 작지 않은 작업 비율 | >= 90% | 작업당 예상 소요 시간 1-4시간 범위 |
| 의존성 정확도 | 실제 의존성과 정의된 의존성 일치율 | >= 98% | `(정확한 의존성 / 전체 의존성) * 100` |
| 누락 작업률 | 추후 추가된 누락 작업 비율 | <= 3% | `(추가 작업 수 / 초기 작업 수) * 100` |

## 2. 조율 효율성 (Coordination Efficiency)

### 2.1 병렬화 효율 (Parallelization Efficiency)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 병렬 작업 비율 | 동시 진행 가능한 작업의 실제 병렬 실행률 | >= 80% | `(병렬 실행 작업 / 병렬 가능 작업) * 100` |
| 유휴 시간 (Idle Time) | 에이전트가 작업 없이 대기한 시간 | <= 10% | 전체 작업 시간 대비 대기 시간 |
| Worktree 활용률 | 독립 작업에 Worktree 사용 비율 | >= 90% | `(Worktree 사용 작업 / 독립 작업) * 100` |

### 2.2 병합 충돌 관리 (Merge Conflict Management)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 충돌 발생률 | 병합 시 충돌 발생 비율 | <= 5% | `(충돌 발생 병합 / 전체 병합) * 100` |
| 충돌 해결 시간 | 충돌 발생 시 해결까지 소요 시간 | <= 30분 | 충돌 감지부터 해결까지 시간 |
| 예방적 조율 성공률 | 사전 조율로 충돌 예방한 비율 | >= 70% | 잠재적 충돌 대비 예방 성공률 |

## 3. 의사결정 품질 (Decision Quality)

### 3.1 브레인스토밍 효과성 (Brainstorming Effectiveness)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 요구사항 커버리지 | 브레인스토밍에서 도출된 요구사항 반영률 | >= 95% | `(반영된 요구사항 / 전체 요구사항) * 100` |
| 설계 변경률 | 구현 중 설계 변경이 필요한 비율 | <= 10% | `(변경된 설계 / 초기 설계) * 100` |
| 위험 식별률 | 사전에 식별한 위험 대비 실제 발생 위험 | >= 80% | `(사전 식별 위험 / 전체 발생 위험) * 100` |

### 3.2 우선순위 결정 정확도 (Priority Accuracy)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| Critical 작업 적시 완료율 | Critical 우선순위 작업의 기한 내 완료율 | 100% | `(기한 내 완료 / Critical 작업 수) * 100` |
| 우선순위 변경률 | 작업 진행 중 우선순위 변경 비율 | <= 10% | `(변경된 작업 / 전체 작업) * 100` |
| 블로커 조기 식별률 | 블로킹 이슈 사전 식별 비율 | >= 85% | `(사전 식별 블로커 / 전체 블로커) * 100` |

## 4. 프로젝트 관리 지표 (Project Management Metrics)

### 4.1 일정 관리 (Schedule Management)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 예상 vs 실제 시간 정확도 | 예상 소요 시간과 실제 소요 시간 차이 | <= 20% | `abs(예상 - 실제) / 예상 * 100` |
| 미션 완료율 | 계획된 미션의 성공적 완료 비율 | >= 95% | `(완료 미션 / 계획 미션) * 100` |
| 지연 발생률 | 예상 일정 대비 지연 발생 비율 | <= 15% | `(지연 작업 / 전체 작업) * 100` |

### 4.2 리소스 관리 (Resource Management)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 에이전트 부하 균형 | 에이전트별 작업량 편차 | <= 20% | 표준편차 / 평균 * 100 |
| 과부하 발생률 | 에이전트가 과부하 상태인 시간 비율 | <= 5% | 동시 작업 3개 이상 시간 |
| 의존성 대기 시간 | 의존성으로 인한 평균 대기 시간 | <= 1시간 | 의존 작업 완료까지 대기 시간 |

## 5. 커뮤니케이션 효과성 (Communication Effectiveness)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 지시 명확성 | 추가 설명 요청 없이 이해된 작업 비율 | >= 90% | `(명확한 지시 / 전체 지시) * 100` |
| 상태 업데이트 정확도 | 보고된 상태와 실제 상태 일치율 | >= 98% | `(정확한 보고 / 전체 보고) * 100` |
| 에스컬레이션 적시성 | 문제 발생 시 적시 에스컬레이션 비율 | >= 95% | 심각도 대비 에스컬레이션 타이밍 |

## 개선 액션 트리거

| 지표 | 임계값 | 트리거 액션 |
|------|--------|-------------|
| 재할당률 > 10% | Critical | 작업 분석 프로세스 점검 |
| 충돌 발생률 > 10% | High | Worktree 전략 재검토 |
| 예상 시간 오차 > 30% | Medium | 추정 방법론 개선 |
| 에이전트 과부하 > 15% | High | 작업 분배 알고리즘 조정 |
