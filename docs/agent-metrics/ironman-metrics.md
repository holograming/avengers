# IronMan 에이전트 성공 메트릭

> 풀스택 개발자(Full-Stack Developer) 역할의 핵심 성과 지표

---

## 1. 코드 품질 (Code Quality)

### 1.1 정적 분석 지표 (Static Analysis Metrics)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 린트 오류 (Lint Errors) | ESLint/TSLint 오류 수 | 0 | CI 린트 검사 결과 |
| 린트 경고 (Lint Warnings) | ESLint/TSLint 경고 수 | <= 5/1000 LOC | 경고 수 / 코드 라인 수 |
| 코드 복잡도 (Cyclomatic Complexity) | 함수당 순환 복잡도 | <= 10 | SonarQube 또는 ESLint |
| 인지 복잡도 (Cognitive Complexity) | 함수당 인지 복잡도 | <= 15 | SonarQube |
| 중복 코드율 (Code Duplication) | 중복된 코드 블록 비율 | <= 3% | `(중복 라인 / 전체 라인) * 100` |

### 1.2 타입 안정성 (Type Safety)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| TypeScript strict mode | strict 모드 활성화 상태 | 100% | tsconfig.json 검사 |
| any 타입 사용률 | any 타입 사용 비율 | <= 1% | `(any 사용 / 전체 타입) * 100` |
| 타입 커버리지 (Type Coverage) | 타입이 명시된 코드 비율 | >= 95% | type-coverage 도구 |

### 1.3 코드 리뷰 품질 (Code Review Quality)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 첫 리뷰 승인률 (First-Pass Approval) | 첫 리뷰에서 승인되는 비율 | >= 80% | `(첫 승인 / 전체 리뷰) * 100` |
| Critical 이슈 발생률 | 리뷰에서 Critical 이슈 발견 비율 | <= 5% | `(Critical 이슈 PR / 전체 PR) * 100` |
| 리뷰 반복 횟수 | PR당 평균 리뷰 라운드 수 | <= 2회 | 승인까지 리뷰 횟수 |

## 2. 테스트 커버리지 (Test Coverage)

### 2.1 커버리지 지표 (Coverage Metrics)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 라인 커버리지 (Line Coverage) | 테스트된 코드 라인 비율 | >= 80% | Jest/Istanbul |
| 브랜치 커버리지 (Branch Coverage) | 테스트된 분기 비율 | >= 75% | Jest/Istanbul |
| 함수 커버리지 (Function Coverage) | 테스트된 함수 비율 | >= 90% | Jest/Istanbul |
| 신규 코드 커버리지 | 새로 작성된 코드의 커버리지 | >= 90% | diff coverage |

### 2.2 테스트 품질 (Test Quality)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 테스트 대 코드 비율 | 테스트 코드 / 프로덕션 코드 | >= 0.8 | LOC 비율 |
| 테스트 실행 시간 | 전체 테스트 실행 소요 시간 | <= 5분 | CI 테스트 시간 |
| 스냅샷 테스트 비율 | 스냅샷 테스트가 차지하는 비율 | <= 20% | 스냅샷 테스트 / 전체 테스트 |

## 3. 빌드 성공 (Build Success)

### 3.1 빌드 안정성 (Build Stability)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 빌드 성공률 (Build Success Rate) | 성공적인 빌드 비율 | >= 98% | `(성공 빌드 / 전체 빌드) * 100` |
| 빌드 시간 (Build Time) | 평균 빌드 소요 시간 | <= 3분 | CI 빌드 로그 |
| 빌드 실패 복구 시간 (MTTR) | 빌드 실패 후 복구까지 시간 | <= 30분 | 실패 감지 ~ 성공 빌드 |

### 3.2 번들 품질 (Bundle Quality)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 번들 크기 (Bundle Size) | 프로덕션 번들 총 크기 | <= 500KB (gzip) | webpack-bundle-analyzer |
| 번들 크기 증가율 | PR당 번들 크기 증가 | <= 5% | `(신규 크기 - 기존 크기) / 기존 크기 * 100` |
| 트리 셰이킹 효율 | 사용되지 않는 코드 제거율 | >= 90% | 빌드 분석 도구 |

## 4. 프론트엔드 성능 (Frontend Performance)

### 4.1 Core Web Vitals

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| LCP (Largest Contentful Paint) | 최대 콘텐츠 렌더링 시간 | <= 2.5s | Lighthouse |
| FID (First Input Delay) | 첫 입력 지연 시간 | <= 100ms | Lighthouse |
| CLS (Cumulative Layout Shift) | 누적 레이아웃 이동 | <= 0.1 | Lighthouse |
| Lighthouse 성능 점수 | 전체 성능 점수 | >= 90 | Lighthouse |

### 4.2 런타임 성능 (Runtime Performance)

| 지표 | 설명 | 목표값 | 측정 방법 |
|------|------|--------|----------|
| 메모리 누수 (Memory Leaks) | 감지된 메모리 누수 수 | 0 | Chrome DevTools |
| 리렌더링 횟수 | 불필요한 리렌더링 비율 | <= 10% | React Profiler |
| API 응답 처리 시간 | 프론트엔드에서 API 응답 처리 시간 | <= 50ms | Performance API |

| 테스트 선행 비율 | 테스트가 구현보다 먼저 작성된 비율 | >= 80% | 커밋 타임스탬프 |
| 리팩토링 빈도 | TDD 사이클당 리팩토링 실행 비율 | >= 70% | 리팩토링 커밋 분석 |

## 개선 액션 트리거

| 지표 | 임계값 | 트리거 액션 |
|------|--------|-------------|
| 커버리지 < 70% | Critical | 코드 동결, 테스트 보강 |
| 빌드 실패률 > 5% | High | 빌드 파이프라인 점검 |
| 번들 크기 증가 > 10% | Medium | 번들 최적화 작업 |
| Lighthouse 점수 < 80 | High | 성능 최적화 스프린트 |
| 리뷰 반복 > 3회 | Medium | 코드 가이드라인 재검토 |
