# Start Mission

새로운 임무를 시작합니다.

## 임무 시작 프로세스

1. **요청 분석** (Captain)
   - 사용자 요청 파악
   - 목표 정의

2. **전략 수립** (Dr.Strange)
   - 접근 방법 시뮬레이션
   - 최적 경로 선택

3. **정보 수집** (Jarvis)
   - 필요한 정보 리서치
   - 기술 문서 수집

4. **태스크 분배** (Captain)
   - 작업 분해
   - 에이전트 할당

5. **실행** (IronMan, Natasha)
   - TDD 기반 개발
   - 워크트리 병렬 작업

6. **검증** (Groot)
   - 테스트 실행
   - 품질 보증

7. **문서화** (Vision)
   - 결과 문서화
   - API 문서 작성

## 사용법

```
/mission {mission_description}
```

예시:
```
/mission 사용자 인증 시스템 구현
/mission REST API 성능 최적화
/mission 다크모드 기능 추가
```

## 인수

$ARGUMENTS - 임무 설명

Captain이 이 임무를 분석하고 팀에게 배분합니다.
